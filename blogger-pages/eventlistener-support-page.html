EVENTLISTENER Support

Welcome to the support page for EVENTLISTENER, a Chrome extension that helps users monitor live tab and page activity, then run selected actions when configured rules are matched.

This page explains how to get started, how to configure rules, what each major feature does, how to troubleshoot common issues, and how to contact support.

Getting Started

  1. Install EVENTLISTENER from the Chrome Web Store.
  2. Pin the extension to your Chrome toolbar if you want quick access.
  3. Open the EVENTLISTENER popup from the toolbar.
  4. Open the full dashboard from the popup.
  5. Configure your saved default rules in the dashboard.
  6. Select the event groups you want to monitor.
  7. Choose the conditions for each selected event group.
  8. Select the action or actions that should run when a rule fires.
  9. Preview and choose an alarm sound if you are using alarm actions.
  10. Configure shortcuts if you want EVENTLISTENER to run a shortcut on the monitored page.
  11. Save your setup.
  12. Return to the page you want to monitor.
  13. Open the EVENTLISTENER popup and click Arm Current Tab.

Basic Workflow

EVENTLISTENER works best when you follow this workflow:

  1. Prepare rules in the dashboard. Choose your event groups, conditions, actions, cooldowns, alarm sound, and shortcut settings.
  2. Arm a tab from the popup. Apply your saved default setup to the current tab.
  3. Let the tab run. EVENTLISTENER watches the selected tab activity while you continue your work.
  4. Review live coverage. Use the dashboard to see which tabs are armed.
  5. Disarm when finished. Stop monitoring manually, or configure a rule action to disarm automatically after firing.

Popup Guide

The popup is the fastest way to use EVENTLISTENER after you have configured your defaults.

Current Tab

The current tab area shows information about the tab you are viewing. This helps you confirm which page you are about to arm or disarm.

Monitoring Status

The monitoring status area shows whether the current tab is armed. If the tab is not armed, EVENTLISTENER is not actively monitoring it with your saved setup.

Arm Current Tab

This applies your saved default rules to the current tab. Use this after you have configured the dashboard and are ready to monitor a page.

Disarm Current Tab

This stops monitoring the current tab. Use it when you no longer need EVENTLISTENER to watch that page.

Test Alarm

This lets you test the selected alarm behavior so you can confirm that sound output is working.

Stop Alarm

This stops an active alarm if one is playing.

Open The Full Dashboard

This opens the main configuration area where you can manage saved defaults, conditions, actions, sounds, shortcuts, cooldowns, and monitored tabs.

Dashboard Guide

The dashboard contains the full EVENTLISTENER configuration experience.

Saved Defaults

Saved defaults are the rules that the popup applies when you arm a tab. Configure these first if you want a repeatable setup.

Rule Editor

The rule editor lets you choose event groups, set conditions, and decide what should happen when those conditions are met.

Trigger Actions

Trigger actions are the things EVENTLISTENER runs when a rule fires. You can enable one action or multiple actions.

Cooldown

Cooldown controls help prevent repeated triggers from firing too frequently. If a page changes rapidly, cooldown settings can reduce noise and make alerts easier to manage.

Live Coverage

The live coverage view shows which tabs are currently armed and which rules are active. Use this view to review, focus, or disarm monitored tabs.

Event Groups Explained

Tab Activity

Use tab activity monitoring for URL changes, title changes, or long loading states. This can help when a web app changes status through the title, redirects to a new URL, or remains in a loading state longer than expected.

Audio And Media

Use audio and media monitoring when sound activity matters. EVENTLISTENER can help detect audio activity or silence conditions depending on your configured rule.

Clicks

Use click monitoring for click inactivity or click bursts. This may be useful for pages where repeated clicking, lack of clicking, or user interaction timing matters.

Scrolling

Use scrolling monitoring to detect scroll inactivity or scroll depth. This may be useful for long pages, feeds, review workflows, documentation, or dashboards.

Keyboard

Use keyboard monitoring for typing inactivity or typing bursts. This can help with pages where text input or keyboard interaction is part of the workflow.

Visibility

Use visibility monitoring to detect when a page stays visible or hidden for a selected amount of time.

DOM Changes

Use DOM change monitoring when page content changes without a full reload. This is useful for dashboards, queues, live feeds, dynamic tables, and web apps that update in place.

Actions Explained

Play Alarm Sound

This plays the selected alarm sound when a rule fires. Use the preview option in the dashboard to test the sound before saving.

Show Notification

This shows a browser notification when a rule fires. If notifications do not appear, check Chrome notification permissions and your operating system notification settings.

Run Shortcut

This sends a configured keyboard shortcut into the monitored page. Shortcut behavior depends on the website. Some websites accept scripted keyboard events, while others ignore or block them.

Stop Screen Share

This attempts to stop an active screen share from the monitored page when supported. The success of this action depends on the browser, page, and screen-sharing context.

Close Tab

This closes the monitored tab when the rule fires. Use this carefully on pages where unsaved work may exist.

Disarm Tab

This stops monitoring after a rule fires. It is useful when you want a rule to run once and then stop.

Troubleshooting

The extension is not detecting activity

  • Make sure the current tab is armed.
  • Open the dashboard and confirm that the event group is enabled.
  • Check that the rule condition matches the behavior you expect.
  • Try testing on a normal website rather than a Chrome internal page.
  • Refresh the page, arm the tab again, and test the condition.
  • Check whether the page restricts extension scripts or injected behavior.

The popup says the tab is not armed

Open the popup on the tab you want to monitor and click Arm Current Tab. If you switch to a different tab, that other tab may not be armed unless you arm it separately.

The alarm does not play

  • Open the dashboard and preview the selected alarm sound.
  • Make sure your device volume is turned up.
  • Check that Chrome is not muted in your operating system sound settings.
  • Use the popup's alarm test control.
  • Try selecting a different alarm sound.

Notifications do not appear

  • Make sure browser notifications are allowed.
  • Check your operating system notification settings.
  • Disable focus mode, do not disturb mode, or notification blocking if needed.
  • Confirm that the notification action is enabled in your rule setup.

The shortcut action does not work

  • Make sure the shortcut is configured correctly in the dashboard.
  • Test the shortcut manually on the target page.
  • Remember that shortcut actions are intended for in-page shortcuts.
  • Some websites may ignore injected keyboard events.
  • Chrome internal pages and restricted pages may not support shortcut injection.

The tab does not close

  • Confirm that the close tab action is enabled.
  • Make sure the rule actually fired.
  • Check whether the tab is a restricted browser page.
  • Try the same setup on a normal web page.

Screen share does not stop

  • Confirm that the stop screen share action is enabled.
  • Check whether the monitored page supports the required behavior.
  • Try ending the screen share manually to confirm browser behavior.
  • Some pages and browser contexts may not allow this action.

Rules fire too often

  • Increase the cooldown period.
  • Make the condition more specific.
  • Disable event groups you do not need.
  • Use the disarm action if the rule should only run once.

Rules do not fire when expected

  • Check that the tab is armed.
  • Check that the selected event group is enabled.
  • Review the condition timing.
  • Test with a simpler condition first.
  • Open the live coverage view to confirm the tab is being monitored.

Best Practices

  • Configure saved defaults in the dashboard before using the popup.
  • Test rules on the actual websites where you plan to use them.
  • Start with one event group, then add more after confirming behavior.
  • Use cooldowns to avoid repeated alerts.
  • Preview alarm sounds before relying on them.
  • Use notifications for low-urgency events and alarms for urgent events.
  • Use disarm-after-fire when you only need a rule to run once.
  • Be careful with close-tab actions on pages where unsaved work may exist.

Permissions And Why They Are Needed

EVENTLISTENER uses Chrome extension permissions to provide its monitoring and automation features. Depending on the version, the extension may use permissions such as alarms, notifications, offscreen, scripting, storage, tabs, and page access.

Alarms Permission

Used for timing, scheduled checks, and cooldown behavior.

Notifications Permission

Used to show browser notifications when a configured rule fires.

Storage Permission

Used to save settings, saved defaults, rule configuration, selected actions, alarm choices, shortcut settings, and monitoring preferences.

Tabs Permission

Used to identify the current tab, monitor armed tabs, and manage tab-related actions.

Scripting Permission

Used to monitor supported page activity and run supported actions in the page context.

Host/Page Access

Used so EVENTLISTENER can operate on the pages you choose to monitor. Some pages may still restrict extension functionality.

Privacy Information

EVENTLISTENER is designed as a browser-based productivity tool. Its purpose is to monitor selected tab activity and run the actions you configure.

Settings and rules are stored using Chrome extension storage. EVENTLISTENER is not designed to sell user data. Always review the Chrome Web Store listing and any published privacy policy for the most current privacy details.

Known Limitations

  • Chrome internal pages may restrict extension behavior.
  • Chrome Web Store pages may restrict extension behavior.
  • Some websites block injected scripts or keyboard events.
  • Notifications depend on browser and operating system settings.
  • Audio behavior can depend on browser policies and system sound settings.
  • Screen-share behavior depends on what the browser and page allow.
  • DOM monitoring may vary depending on how a page updates its content.
  • Shortcut actions are not intended for operating-system shortcuts.

Frequently Asked Questions

Do I need to configure the dashboard first?

Yes, it is recommended. Configure your saved defaults in the dashboard first, then use the popup to quickly arm the current tab.

Can I arm more than one tab?

Yes. EVENTLISTENER is designed to support live monitoring workflows where multiple tabs may be armed and reviewed from the live coverage view.

Can I stop monitoring a tab?

Yes. Use the popup to disarm the current tab or use the dashboard live coverage view to review and disarm monitored tabs.

Can a rule run more than one action?

Yes. You can configure multiple trigger actions for a rule.

Can I test the alarm sound?

Yes. Use the alarm preview controls in the dashboard or the alarm test control in the popup.

Does EVENTLISTENER work on mobile Chrome?

EVENTLISTENER is intended for desktop Chrome and Chromium-based browsers that support the required Chrome extension APIs.

Why does the extension need access to websites?

EVENTLISTENER needs page access so it can monitor activity on the tabs you choose to arm and run supported actions when rules fire.

Does EVENTLISTENER change web pages by itself?

EVENTLISTENER only runs the actions you configure. Some actions, such as shortcut actions, are intended to interact with the monitored page.

What should I do before reporting a bug?

Try refreshing the page, re-arming the tab, checking your saved defaults, and testing on a normal website. If the issue continues, contact support with the details listed below.

Uninstalling EVENTLISTENER

  1. Open Chrome.
  2. Go to chrome://extensions/.
  3. Find EVENTLISTENER.
  4. Click Remove.
  5. Confirm removal.

Removing the extension will remove the extension from Chrome. Depending on Chrome behavior and sync settings, extension-related local settings may also be removed.

Safety And Responsible Use

Use EVENTLISTENER only on pages and workflows where you understand the effects of the actions you configure. Be careful with actions such as closing tabs, running shortcuts, or stopping screen share. Test rules before relying on them for important workflows.
